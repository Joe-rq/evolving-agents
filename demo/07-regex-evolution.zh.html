从 .* 到精确正则 可观察进化 程序合成任务

正则合成领域 · 生成器从成功模式嫁接收紧约束 · 配 experiments/regex_evolution.py。与符号回归同构，但目标是程序合成（从样本归纳规则）。

目标一：从样本归纳日期规则 — F1 跨轮轨迹

隐藏规则 YYYY-MM-DD （seed=3）。给 agent 一组正例（该匹配）和反例（不该匹配），让它归纳出规则。.* 匹配一切（recall 满但 precision 低）→ F1 中等。收紧后 F1 爬升。
完整组（memory-aware 嫁接收紧） 基线组（memory-independent）
一条进化链：
0.58 [0-9a-z].+$ ← r2 弱线索（锁了数字+尾锚，但仍太宽）
↓ r3：生成器读 Memory，发现该线索 → 结构替换：通配 blob → 三段数字序列
0.74 [0-9]+-[0-9]+-[0-9]+ graft: 整体替换为结构化序列 → F1 跃升
↓ r6-r7：在成功模式上逐步加锚收紧（+^, +$）
0.74 ^[0-9]+-[0-9]+-[0-9]+$ graft: +^ +$（双锚锁定位置）
进化 = 从太宽到刚好。 .*（太宽，F1=0.58）经结构替换变成 [0-9]+-[0-9]+-[0-9]+（F1=0.74），再逐步加锚收紧。这是程序合成的进化形状——不是调参，是模式的表达力在长出来。

目标二：归纳手机号规则 — 证非 cherry-pick

隐藏规则 中国手机号（11位、1开头） （seed=1）。换目标 + 换 seed，排除"只挑了好跑的"。
0.71 \w+\d.{3}$ ← r0 随机池弱线索
↓ r1：结构替换 → 发现"1 开头"这个核心特征
0.77 1[0-9]+[0-9]{4}[0-9]+$ graft: 替换 + 收紧 → 锁定"1开头 + 数字 + 尾锚"
两个目标，同一种模式：完整组从弱线索出发，经结构替换跃升，再逐步收紧。基线组在两个目标上都崩盘（F1→0，不会从成功学）。换目标、换 seed，模式一致。

诚实边界

✅ 证明了⚠️ 局限
同一套进化机制（引擎不改），在程序合成任务（正则归纳）上也产生可观察的进化：F1 从 ~0.5 爬到 0.74-0.77。 F1 峰值（0.74-0.77）低于符号回归（0.99）——正则的结构替换比公式嫁接更高方差（替换对了跃升，错了归零）。
这是第四个领域（量化·2048·符号回归·正则），引擎一行未改接入。进化形式不同（收紧 vs 嫁接），但"从成功里学"一致。 仍是种子化轨迹，非 M-复制对照。正则的 graft 高方差让某些 seed 不稳定（email 目标尤其）。
定位：正则合成不是"最强进化案例"（那是符号回归），它是"进化机制跨任务类型泛化"的证据——从数学公式发现（symreg）到程序合成（regex），同一套引擎都能展示可观察的进化。这比"换了个差不多的数学目标"更有说服力。
