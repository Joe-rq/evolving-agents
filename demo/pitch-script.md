## 段 1 · 钩子（30s）

> "我对量化一窍不通。但我训了一个 agent，它自己学会了：在哪个市场挖、挖哪类信号、避开它过去失败过的方向。它产出了 5 条可提交的 alpha——比我这个不懂量化的主人更懂该怎么挖。"

**要点**：把"不懂量化"从弱点变成 agent 价值的证明。挡住后续所有量化追问——"我不关注 alpha 的金融表现，我关注 agent 怎么学会换池塘的"。

---

## 段 2 · 痛点（40s）

> "自进化 agent 这个赛道很热。但有个问题——大多数 demo 给你一个三天学习曲线、一堆 spectacular 计数，却没有对照实验，也不披露边界。你没法分辨：这是真的进化，还是 in-context learning？是 agent 学的，还是 cherry-picking？
>
> 我们做这套东西时，自己设计过失忆组 vs 完整组的对照。结论是：**跑不了**——机制上线太晚，数据太少。所以我们换了一条路：不吹对照证明，讲一个真实的、可追溯的进化事件。"

**要点**：把"没对照"从弱点转成诚实优势。QuantML 吹不出的，正是这个边界。

---

## 段 3 · 核心证据（60s）— 切图 1

> "看这条因子。它在两个市场跑过。
>
> TOP3000，大盘——self_corr 0.78，太拥挤，失败。
> TOP1000，中小盘——self_corr 0.68，通过，上线了。
>
> 关键看这里：**Sharpe 两边都是 2.15，一模一样。** 因子的质量没变。变的只有拥挤度。所以问题不在因子，在池塘。
>
> agent 没有让我调 decay、调窗口——那永远修不好拥挤。它提议的是换池塘。我批准了。这是 agent 提议、人批准，不是全自动。"

**要点**：图 1 是脸。"Sharpe 没变 / 拥挤度变了"是 Factor 4 最干净的例证。诚实标注人机协同。

---

## 段 4 · 怎么走到（50s）— 切图 2

> "这次换池塘不是碰运气。它是一条学习链的终点。
>
> 6 月 23 号引擎上线，还是冷启动。24 号，agent 学会不再重测已知死因子——这是 Factor 1。同一天，一次失败后它降权了整个 close-yield motif 类，不只屏蔽那一条——Factor 3。然后它把记忆按池塘隔离，TOP3000 的失败不污染 TOP1000——这让'换池塘'变得安全可考虑。最后，agent 产出 evolution action：'别调窗口，换数据源'，提议换到 TOP1000。
>
> 每个节点都是真实 commit。学习是 agent 自己做的，换池塘是 agent 提议、我批准的。"

**要点**：图 2 证明"不是魔法"。commit hash 可追溯。强调 agent-proposes/human-approves。

---

## 段 5 · 为什么可信（50s）— 切图 3

> "为什么你们该信这是真的进化，不是黑箱？
>
> 看这条规则。引擎屏蔽一个候选时，能回答'为什么'。这条 cashflow_close_yield 的 block 规则，分数调整 -0.65，置信度 0.75——追下去，是这 3 条具体的失败表达式，每条都标了 SELF_CORR_FAIL。
>
> 这条规则不是人写的。它是 agent 从账本投影出来的。每条惩罚带 evidence_ids，能追到具体哪几次失败教的。
>
> 这是和只会吹学习曲线的系统之间的差别——它们回答不了'哪几次失败教的'。"

**要点**：图 3 是差异化武器。evidence_ids 可审计 = QuantML 没有的。不点名 QuantML，说"这类系统"。

---

## 段 6 · 这是什么（40s）— 切图 4

> "这套机制我们抽成了一个开源引擎——evolving-agents。
>
> 通用核心实现五个 factor，不绑定任何领域。换领域只换适配器——适配器填领域知识，引擎一行不改。量化只是我们验证它的第一个沙盘。
>
> 有人会问和 EvoMap 什么区别。EvoMap 是 A2A 跨节点经验共享的平台，我们用过它的 memory 层。evolving-agents 是本地、可嵌入的引擎——不同抽象层，更简单，五个命名的 factor 可教。"

**要点**：图 4 定位。正面回答 EvoMap 撞车——平台 vs 引擎，不同层。承认用过 EvoMap（诚实）。

---

## 段 7 · 诚实收尾（30s）

> "我不 claim 这个机制被对照证明有效——我们设计了对照，但数据不够，跑不了。我 claim 的是：五个机制全部实现，有一次 agent 提议换池塘救活死因子的真实事件，每条惩罚可追溯。
>
> 有效性的对照证明在 roadmap 上。我们公开承认这一点——因为另一种选择是营销。"

**要点**：边界内原句收尾。诚实是差异化，不是软弱。

---

## 三记质疑话术（Q&A 备用）

### Q1："这是策略系统，不是 agent 吧？"

> "回测器跑人写的策略。这个 agent 不一样——它自主决定挖哪个方向（Factor 2，给搜索策略打分），自主提议换池塘（Factor 4）。换池塘那个认知跳跃——'不是因子死，是池塘错'——是 agent 做出的，我只是批准。回测器不会自己想到换市场。"

### Q2："它到底学的什么？"

> "它学的不是某个具体 alpha，是**搜索策略本身**——往哪个经济族搜、选哪些字段、避开哪些失败模式。这是策略空间上的元学习，比学某个解高一个层次。换池塘就是它改进自己搜索方法的一次实例。"

### Q3："这量化项目都是 agent 干的，你的贡献在哪？"

> "量化沙盘是 agent 跑的——我不懂量化，正是 agent 帮我跨过了领域门槛。但我的核心作品不是量化策略，是**让 agent 能自进化的那套机制**：research-state 记忆、evolution 学习、motif 类降权、universe 隔离。这些是工程设计，是我做的。沙盘是 agent 跑的，机制是我设计的——这个边界很清楚。"

## 演讲注意事项

- **不要说"自动进化"**——全程用"agent 提议 / 人批准"。诚实边界是卖点。
- **不要念数字**——图 1 的 2.15 / 0.78 / 0.68 让图自己说话，口播只点"没变/变了"。
- **被问量化细节就引回机制**——"这正是关键，我不关注这个 alpha 的金融表现，我关注 agent 怎么学会换池塘的。"
- **三记质疑要练熟**——这是评委必问，磕巴就露怯。
- **图 3 的 evidence_ids 是杀手锏**——如果时间紧，图 2 可以快讲，图 3 不能省。
