# 案例研究：迷宫领域 — 一个没能展示进化的领域，以及我们为什么如实报告

> 这不是"成功案例"。这是一个我们**试图**让进化在迷宫里可见、验证了四条路全部证伪、最终诚实保留代码但降级 claim 的领域。
>
> 为什么仍然值得有一份 case study？因为**记录"什么不 work 和为什么"本身就是诚实差异化**。大多数"自进化 agent"demo 只展示成功的领域，从不告诉你它在哪里失败了。我们公开失败。

## 这个领域要证明什么

引擎一行未改，能不能接入第二个全新领域？（2048 是第一个公开领域，迷宫是第二个。）以及——它能像符号回归那样展示可观察的进化吗？

第一个问题答案是**是**。第二个答案是**否**——四个方向全部证伪。本文记录这个过程。

## 领域设计

迷宫路径规划。一个候选是一段**离散组合策略**（不是连续权重，这是和 2048 的刻意对比）：

| 槽位 | 可选值 | 含义 |
|---|---|---|
| `explorer` | wall_follow / greedy_dist / flood / random | 探路流派 |
| `dead_end` | backtrack / pivot_turn / random | 撞墙反应 |
| `junction` | left_hand / right_hand / by_distance | 分叉选择 |
| `marking` | none / visited_averse | 标记策略 |

总组合数：4 × 3 × 3 × 2 = **72 个**。

引擎完整接入（5 协议 + RuleSpec + execute callback + memory-aware 生成器），代码见 `adapters/maze/`。**这一步本身是成功**——证明领域无关性不是空话。

## 为什么进化在这里不成立——四条路

### 路径 1：离散槽位重组（失败）

最初的设计："成功家族外推" = agent 发现 greedy_dist 成功 → 围绕它重组其他槽位。

**实测**：`flood` 家族 100% 高效到达（BFS 距离是完美启发式），一上来就满分，没有爬升空间。而 `greedy_dist` 家族内部，不管其他槽位怎么组合，效率要么 0.954 要么 0（到不了）——**离散槽位组合造不出连续梯度**。

### 路径 2：连续 noise 参数（失败）

给决策加连续 `noise`（以概率 p 随机选方向），让策略空间不可穷举。

**实测**：到达率随 noise **上升**（0%→75%），但效率全程极低（<0.1）。更反直觉的是：noise=0（纯贪心）到达率 0%——因为贪心会撞死胡同，要靠随机扰动才能逃离。**迷宫的"成功"靠随机性，不靠"策略越来越好"**——和进化背道而驰。

### 路径 3：资源约束 cost 梯度（部分成功，但不够）

这条**打开了真实的梯度**：给 `flood`（每步算 BFS，O(n²)）加计算代价，让 `greedy_dist`（manhattan，O(1)）凭"准且省"获胜。

| explorer | 步数/最优 | 计算代价/步 | 总代价（α=1.0）|
|---|---|---|---|
| **greedy_dist** | 1.01x | O(1) | **1.07**（最低）|
| flood | 1.00x | O(n²) | 1.98 |
| wall_follow | 2.22x | O(1) | 2.29 |
| random | 2.45x | O(1) | 3.16 |

`flood` 不再是无悬念赢家。这是迷宫**第一次出现进化空间**。但——

### 路径 4：cost-aware 生成器（失败）

基于路径 3 的梯度，重写生成器，让它从**低成本**成功候选外推。

**实测**：完整组 vs 基线组**无稳定分化**。frugal-success 比例和 avg_cost 互有高低，无趋势。三重原因：

1. **greedy_dist 内在不稳定**：同一策略时而 cost=1.02（最优）时而 99（撞死），信号被噪声淹没。
2. **72 个组合几轮饱和**：生成器没有"还没探索的省力方向"可外推。
3. **play 的随机 tie-break** 让每轮高方差，单轮 5 个样本撑不起统计信号。

## 根因：领域属性 vs 工程努力

这不是"再调调参就行"的问题。迷宫是**组合搜索**，它的难度来自**信息缺失**（看不见全图），而非"策略表达力不足"。增加策略表达力（连续参数、资源约束、cost-aware 生成器）都改变不了这个本质。

对比符号回归：R² 是**确定性函数**（同一公式→同一 R²），所以进化信号干净；迷宫的 play 有随机性 + 策略不稳定，信号被淹没。**领域的选择决定进化是否可见**——这是我们最重要的元教训。

## 这证明了 / 没证明什么

- ✅ **证明了**：引擎一行未改接入第二个全新领域（领域无关性的直接证据）；资源约束能打破"强策略统治"（flood 不再满分，这是一条真实的技术贡献）。
- ❌ **没证明**：迷宫无法展示可观察的进化。四个方向（离散重组 / 连续 noise / 资源约束梯度 / cost-aware 生成器）全部证伪。
- 🧭 **诚实定位**：迷宫的角色是"机制跨领域接入 + 资源约束探索"，**不是**进化案例。进化的主案例是符号回归。我们如实区分——不为迷宫编造进化叙事。

## 为什么仍然保留这份 case study

三个理由：

1. **诚实差异化**：大多数"自进化 agent"demo 只展示成功，从不公开失败。我们公开"什么不 work 和为什么"——这比一条不受控的学习曲线可信。
2. **元教训**：领域选择决定进化可见性。这个洞察本身有价值——它指导了符号回归的成功（选了确定性 R² 的领域）。
3. **工程价值**：资源约束机制（compute 代价追踪 + 总代价评分）是真实代码，可在其他领域复用。

## demo 钩子（如果被问到迷宫）

> "我们也试了迷宫。试了四条路——离散重组、连续参数、资源约束、cost-aware 生成器——全部没跑出可观察的进化。原因是迷宫是组合搜索，难度来自看不见全图，增加策略表达力不解决问题。所以我们没有为它编进化叙事，而是转到了符号回归——一个 R² 有连续梯度的领域。这个'试错 + 如实转向'的过程，是我们和那些只展示成功曲线的系统之间的差别。"
