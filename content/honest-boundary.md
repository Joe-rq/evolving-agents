# 诚实声明边界

> 这个项目对自进化能 claim 什么、不能 claim 什么——以及我们为什么公开披露这个边界。

这是 repo 里最重要的文档。每份 demo、pitch、factor 文档都必须守在这个边界内。

## 我们 claim 的

1. **5 个机制都实现了。** Memory、MetaLearner、失败模式惩罚、结构 pivot、新颖性防护——都作为代码存在，且在一个真实负载上跑过（量化因子挖掘）。

2. **观察到过真实的进化行为。** 最具体的：同一条因子表达式在 TOP3000（大盘，self_corr 0.78 FAIL）**失败**，在 agent 提议换池塘后在 TOP1000（中小盘，Sharpe 2.15，self_corr 0.68 PASS）**复活**。一次真实的换结构事件，有 commit 历史——不是构造出来的 demo。

3. **部分学习是 agent 从账本自作的**，不纯粹是手写规则（比如 close-yield motif 类惩罚，commit `3a38a52`）。

## 我们不 claim 的

1. **有效性对照：做了，结果 null。** 我们在公开的 2048 领域跑了一个预注册的失忆组 vs 完整组对照（M=30 配对复制，Wilcoxon α=0.05）。结果：**p=0.079，未检出显著增益**——机制作为代码可观察地工作，但学习增益未被统计证明。完整协议与结果 → [2048 案例研究](../examples/case-study-2048-ablation.md)。量化领域仍无对照（私有数据量太小）。**"有效性被证明"依然不成立——只是这次是我们自己跑对照、自己报的 null，不是"没做"。**

2. **没有量化的收益。** 2048 对照未检出显著提升；量化领域无对照。任何改善曲线都和 in-context learning、人机协同决策混淆——我们不 claim "进化提升 X%"。

3. **跨领域：初步验证，非泛化证明。** 机制现在在**七个**领域跑过（量化因子挖掘 + 2048 + 符号回归 + 正则合成 + SQL 查询优化 + 代码生成 + RAG 检索），引擎一行未改接上新领域。后三个（SQL + 代码 + RAG）从 Roadmap 变成了已验证。但 7 个领域 ≠ 泛化 claim；领域无关仍是*设计意图 + 早期证据*，不是被证明的普适性。

## 我们为什么公开披露

有几个项目营销"自进化 agent"，带三天学习曲线和 spectacular 计数，**却没有对照实验、也不披露边界**。它们的"进化"无法和 in-context learning 或 cherry-picking 区分。

我们相信诚实的路——*机制 + 观察到的行为 + 披露的边界*——比一个不受控的 claim 更持久。有效性验证在我们的 roadmap 上；它不是今天的 claim。

## 2026-06-30 更新：2048 公开对照（null 结果）

我们在公开的 2048 领域补上了原跑不了的失忆组对照。**结果是 null（p=0.079）**。这是项目至今最硬的一块诚实：

- ✅ 5 个机制在全新公开领域端到端跑通；**机制作为代码可观察地工作**（block 452 次 / pivot 触发 131 次 / 投影规则 1.7 条，失忆组几乎为零）。
- ⚠️ 但累积 memory 带来的执行质量提升**未达统计显著**（中位数 +0.083 log₂，CI [0.000, 0.167]，低于 MDE≈0.10）。

我们区分**「机制存在」**与**「机制有效」**：前者被这次实验再次确认，后者未被检出。我们如实报告 null，不藏起来，也不靠机制层证据暗示增益。→ [2048 案例研究](../examples/case-study-2048-ablation.md) · [预注册协议](../experiments/pre_registration.md)

## 2026-07-01 更新：第三个领域（符号回归）— 可观察的生成器进化

我们新增了**符号回归**领域（`adapters/symreg/`）——不是为了"有效性"统计 claim，而是为了回答 2048 结构上回答不了的一个问题：

> "你所有的改进都来自**屏蔽**死过的东西。屏蔽不是进化。agent 能不能**造出**新的、更好的东西？"

2048 的候选池 memory-independent（为对照纯洁性必须如此），所以引擎只能挑不能造。符号回归换了一条路：**生成器读 Memory，从成功线索嫁接出新公式。** 实验在**两个隐藏目标**（`2·x²+1` seed=3、`x²+sin(x)` seed=7）上跑出了可观察的进化轨迹——例如 `(x*x)`（R²=0.37）经嫁接 `×2` 长出 `2*(x*x)`（R²=0.996），完整组 R² 跨轮稳定在 0.98+，基线组剧烈震荡至崩盘。→ [符号回归案例研究](../examples/case-study-symreg-evolution.md) · [实验脚本](../experiments/symreg_evolution.py)

**诚实边界（新增）**：
- ✅ 可 claim：生成器从成功嫁接出新公式是**可观察的**（结构可见、血统可追溯）；**两个目标 + 不同 seed 模式一致 → 非单例**；这是 2048 结构上做不到的"生成式进化"。
- ⚠️ 不 claim：这是 **2 条种子化轨迹**，不是 M-复制对照实验。"生成器进化带来统计显著提升"未被证明。我们和 2048 一样区分「机制存在」与「机制有效」，不把前者打扮成后者。

**关于迷宫领域（`adapters/maze/`）**：我们**也**试了让进化在迷宫里可见——验证了四条路（离散重组 / 连续参数 / 资源约束梯度 / cost-aware 生成器）全部证伪。根因是迷宫作为组合搜索领域，难度来自信息缺失而非策略表达力不足。我们**没有**为它编造进化叙事，而是如实记录失败并降级为"机制跨领域接入"的证据。→ [迷宫案例研究（失败记录）](../examples/case-study-maze.md)。这个"试错 + 如实转向"的过程本身就是诚实差异化。

## 2026-07-01 更新：第四个领域（正则合成）— 跨任务类型的进化泛化

符号回归（公式发现）证明了"agent 能造新候选"。我们追问：**同一套机制在完全不同的任务类型上是否也成立？** 正则合成（`adapters/regex/`）是程序合成任务——从正例/反例归纳字符串规则（日期、邮箱、手机号）。它的进化方向和符号回归**相反**：不是加性嫁接（公式越长越复杂），而是**收紧替换**（`.*` 太宽 → `[0-9]+-[0-9]+-[0-9]+` 刚好）。

结果：F1 从 ~0.5 爬到 0.74-0.77（日期 `[0-9]+-[0-9]+-[0-9]+`、手机号 `1[0-9]+...`），完整组守住、基线组崩盘。→ [正则合成案例研究](../examples/case-study-regex-evolution.md)

**诚实边界**：F1 峰值（0.74-0.77）低于符号回归（0.99）——正则的结构替换比公式嫁接更高方差。仍是种子化轨迹。但关键意义是**跨任务类型**（公式发现 vs 程序合成）的进化一致——这比"换个差不多的数学目标"更能支撑"机制泛化"。

至此机制在**七个**领域跑通（量化因子挖掘 · 2048 · 符号回归 · 正则合成 · SQL 查询优化 · 代码生成 · RAG 检索），引擎一行未改接入。后三个（SQL + 代码 + RAG）从 Roadmap 变成了已验证：SQL 代价从 24→1（反模式修复链），代码从部分通过到全通过（bug 修复链），RAG 从 0→1（策略配置逐步调优链）。但 7 个领域 ≠ 泛化 claim——领域无关仍是*设计意图 + 早期证据*，不是被证明的普适性。

## 边界内的一句话 pitch

> "五个自进化机制，全部实现，跨三个领域（量化·2048·符号回归）跑通；我们在公开的 2048 领域做了预注册的失忆组对照——结果 null（p=0.079）：机制可观察地工作，但增益未被统计证明。我们还在符号回归领域展示了生成器从成功嫁接出新公式的**可观察进化**——这是 2048 结构上做不到的。我们如实区分「机制存在」与「机制有效」，因为另一种选择是把前者打扮成后者。"
