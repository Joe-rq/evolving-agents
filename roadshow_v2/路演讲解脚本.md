**【口播】**
> 大家好，这个项目叫**痕进**——失败留痕，凭痕而进。英文名 Scarforge。我们做的是：给 AI 装一个**从自己的失败里学习**的能力。

**【语气】** 平稳、自信。不要急。停一秒再翻页。

### 第 2 页 · 承诺（20 秒）

**【操作】** 停在第 2 页（大字"失败留痕，凭痕而进"）。

**【口播】**
> 一句话讲完我们在干什么：**痕进不是让 Agent 背更多上下文，而是让它学会长记性**——错在哪里、为什么错、下次该避开哪条路，都留下可追溯的痕迹。

**【语气】** 这句话要慢、要清楚。这是全场第一句评委能记住的话。

**⟶ 翻到第 3 页**

### 第 3 页 · 问题（20 秒）

**【操作】** 第 3 页（"今天的 Agent 太容易反复犯同一个错"）。

**【口播】**
> 为什么需要这个？因为今天的 Agent 有个通病：**一次失败修掉了，下一次换个壳又回来**。客服、代码、搜索都一样。问题不是记忆不够，是**失败没有进入它的决策系统**。

**【语气】** 这里语气转沉——你在讲痛点，要让评委点头。

**⟶ 翻到第 4 页**

### 第 4 页 · 论点（15 秒）

**【操作】** 第 4 页（"不是修 prompt，是让失败变成系统资产"）。

**【口播】**
> 我们的论点：一次失败如果只停在对话里，就是噪音。但**写进账本、归成模式、投影成规则、影响下一轮搜索**——它才开始产生复利。这就是痕进做的事。

**【语气】** 加快一点节奏，引出核心证据。

**⟶ 翻到第 5 页**

### 第 5 页 · 核心证据：换池塘（60 秒）⭐ 重点

**【操作】** 第 5 页（"交易不是终点，只是第一个沙盘"+ 换池塘对比图）。

**【口播】**
> 来看最硬的一个案例。这是我们训的一个做量化因子挖掘的 Agent。它挖出了一条因子——
>
> **（指左边 TOP3000）** 在大盘里，这条因子 Sharpe 2.15，但 self_corr 0.78——太拥挤了，**失败**。
>
> **（指右边 TOP1000）** 同一条因子，换到中小盘——Sharpe 还是 2.15，但 self_corr 降到 0.68，**通过了，上线了**。
>
> **关键：Sharpe 两边一模一样。因子的质量没变。变的只有拥挤度。** 所以问题不在因子，在池塘。
>
> Agent 没有让我去调参数——那永远修不好拥挤。它**提议的是换池塘**。我批准了。这是 Agent 提议、人批准，不是全自动。

**【语气】** "Sharpe 没变 / 拥挤度变了"这句是全场最关键的 punchline，要重读、要停顿。让数字自己说话。

**【⚠️ 兜底】** 如果被追问量化细节 → "这正是关键，我不关注这个 alpha 的金融表现，我关注 Agent 怎么学会换池塘的。"

**⟶ 翻到第 6 页**

### 第 6 页 · 公开进化：符号回归 + 正则合成（30 秒）⭐ 承上启下

**【操作】** 第 6 页（"私有案例验证不了？换成公开领域"）。

**【口播】**
> 这个案例是私有的，评委没法复现。所以我们补了**两个公开领域**——任何人能跑、能验证。
>
> **（指左边 symreg）** 第一个，符号回归：Agent 猜数学公式。它发现"x 的平方"比较接近，就嫁接了一个系数，从 `(x*x)` 长出 `2*(x*x)`——R² 从 0.37 跃升到 0.996。
>
> **（指右边 regex）** 第二个，正则合成：Agent 猜分类规则。从"什么都框进来"收紧到精确的日期格式——这是程序合成任务，进化的方向和公式相反，但学法一样。

> **同一套引擎，两种不同的任务类型，都能从成功里学。** 接下来我用一个动画让你亲眼看到这个过程。

**【语气】** 这是承上启下的页——从"私有故事"过渡到"公开可复现 + 动画高潮"。语速可以快一点，重点是引出动画。

## 五、Q&A 应对手册

### Q1："这到底是 Agent 还是策略系统？"

> 回测器跑人写的策略。这个 Agent 不一样——它**自主决定**挖哪个方向，**自主提议**换池塘。换池塘那个认知跳跃——"不是因子死，是池塘错"——是 Agent 做的，我只是批准。回测器不会自己想到换市场。

### Q2："对照跑出 null，不就说明没用？"

> 恰恰相反。null 说明机制**作为代码可观察地工作**：完整组屏蔽了 452 个失败候选、触发 131 次结构性 pivot——失忆组几乎全零。没达标的只是"学习增益的统计检出"，不是"机制是否存在"。我们如实区分这两件事——这比给你一条不受控的学习曲线诚实得多。

### Q3："这和 ChatGPT 记住对话有什么区别？"

> ChatGPT 的记忆是**把历史塞进 prompt 碰运气**——我们实测发现这对常规场景零增量。痕进不一样：它把失败**结构化**——归成模式、投影成规则、带证据链。下次不是"回忆"，是"查规则"。这是 RAG 式召回做不到的。

### Q4："动画里那个公式跳跃，是不是你预设好的？"

> 不是。动画的每一轮都是实验脚本 `experiments/symreg_evolution.py` 真跑出来的，数据在 `experiments/_symreg_offset_s3.json` 里，任何人可以复现。公式是 Agent 的生成器从成功线索嫁接出来的，每一步的父代和嫁接操作都有记录——你可以追溯到具体是哪一轮的哪个公式"生"了它。

### Q5："四个领域都这么toy，真实场景呢？"

> 第一个领域就是真实的——量化因子挖掘，私有数据，真实 case。换池塘那个事件有 commit 历史，不是构造的 demo。其他三个公开领域是为了**可复现性**——私有数据评委验证不了，所以我们补了三个任何人能跑的公开领域。真实场景 + 公开可复现，两个都要。

### Q6（杀手锏问题）："你们的核心贡献到底是什么？"

> 三个字：**可审计**。别的系统说"Agent 学到了"，你问"哪几次失败教的"，它答不上来。我们能——每条规则带 evidence_ids，追到具体失败。在一个满是黑箱学习曲线的赛道里，能回答"为什么"就是差异化。

### Q7："这个引擎需要依赖大模型吗？跑起来贵不贵？"

> **引擎本身零 LLM 依赖。** 全部用 Python 标准库，本地跑、离线跑、免费跑。引擎做的事是结构化的记录和检索——记住失败、屏蔽死路、给家族打分、投影规则，全是确定性的数据结构操作，不需要"理解"或"推理"，所以不调任何大模型。
>
> 大多数"自进化 Agent"的进化依赖 LLM 的 in-context learning——贵、不可复现、不可审计。痕进的进化是结构化的——免费、确定、可追溯。在你的项目里，LLM 可以作为候选生成器（负责"造"），但引擎的记忆和筛选（负责"筛和记"）不依赖它。
