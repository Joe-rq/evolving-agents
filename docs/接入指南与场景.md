# 痕进 · 接入指南与适用场景

> 回答三个问题：第一步为什么是搜索、适用于哪些场景、怎么在一个项目里接入使用。

---

## 一、为什么第一步是"搜索"而不是"直接算出答案"？

### 本质：这类问题没有解析解

痕进解决的是**搜索问题**——在巨大的可能空间里找一个"足够好"的解。这类问题的特点是：

- **没有公式可以算出最优解**。你不可能"算出"最好的因子、最好的公式、最好的正则——只能在可能的空间里探索。
- **反馈是有的，但只在试过之后**。一条因子好不好，只有跑完回测才知道；一个公式对不对，只有拟合后才知道。

所以第一步必然是**广撒网**——Memory 是空的，Agent 没有任何经验可参考。这不是缺陷，是搜索问题的**必然起点**。

### 类比：生物进化

基因突变是随机的（第一步"猜"），但自然选择让它逐步收敛（后续轮次"学"）。痕进做的是同一件事——**第一步广撒网是必要的，进化的价值在后续轮次的收敛**：

| 轮次 | Memory 状态 | 行为 | R²（符号回归实例） |
|---|---|---|---|
| 第 1 轮 | 空 | 广撒网，试各种方向 | 0.37（很低，符合预期） |
| 第 2 轮 | 记住了"哪些方向全错" | 屏蔽死路，省下力气 | 0.37（守住线索） |
| 第 3 轮 | 记住了"x² 这个方向有希望" | 在成功方向上嫁接 | **0.996（跃升）** |
| 第 5 轮 | 积累了多条成功线索 | 围绕最佳线索深化 | 0.996（稳定） |

**如果第一步就很准，说明问题太简单——不需要进化。第一步必须"差"，才有进步空间。**

### 关键区分：搜索 vs 优化

| | 优化问题 | 搜索问题（痕进的领域） |
|---|---|---|
| 有公式吗 | 有（梯度下降） | 没有（空间是离散/组合的） |
| 第一步 | 从初始点沿梯度走 | 广撒网（没有梯度可沿） |
| 怎么变好 | 调参数（连续空间） | 记住失败 + 在成功上生长（离散空间） |

2048 失败的部分原因就是它被当成了优化问题（连续权重），但实际是搜索问题。符号回归成功是因为它正确地被当作搜索问题处理（离散 AST）。

## 二、适用于哪些场景？

### 判断标准：三个必要条件

一个场景适合用痕进，必须**同时满足**：

1. **Agent 会反复试错**——不是一次性任务，而是多轮搜索
2. **失败能被明确定义**——有客观的"行/不行"标准，不是主观评价
3. **同类错误会重复出现**——否则记一次就没用了

### 真实场景（按成熟度排序）

| 场景 | 试错循环 | 失败定义 | 痕进的价值 | 成熟度 |
|---|---|---|---|---|
| **量化因子挖掘** | 提因子→回测→看指标 | Sharpe 低、拥挤度高 | 记住"这类信号族在这个市场失效"，换方向不调参 | ✅ **已验证**（私有，真实案例） |
| **符号回归** | 猜公式→拟合→看 R² | R² 低 | 记住"三角函数族拟合不了多项式"，换结构 | ✅ **已验证**（公开） |
| **正则/规则合成** | 猜规则→分类→看 F1 | 假阳性/假阴性 | 记住"过宽的规则会误伤"，收紧 | ✅ **已验证**（公开） |
| **代码生成** | 写代码→编译测试→修 | 编译错误、测试失败 | 记住"这类 API 调用容易触发异常"，换写法 | 🔜 Roadmap |
| **SQL 查询优化** | 写查询→执行→看性能 | 超时、全表扫描 | 记住"这类 JOIN 在大表上必慢"，换路径 | 🔜 Roadmap |
| **RAG 检索** | 检索→生成→评估 | 幻觉、答非所问 | 记住"这类查询召回总是差"，进化检索策略 | 🔜 Roadmap |
| **自动化测试生成** | 生成测试→运行→覆盖率 | 漏边界、没覆盖 | 记住"这类输入组合容易藏 bug"，针对性生成 | 🔜 Roadmap |

### 不适合的场景

- **一次性任务**（写一封邮件）——没有反复试错
- **主观评价**（文章写得好不好）——失败无法定义
- **完全新颖每次不同**（每次都是全新问题）——记了也不重复

## 三、怎么在一个项目里接入？

### 核心设计：引擎不改，只写 adapter

痕进的架构是**两层分离**：

```
你的项目
  │
  ├── 你的 Agent（负责做事：调 API、跑代码、查数据库）
  │
  ├── 痕进 engine（负责进化：记失败、学模式、更新搜索）
  │      └── core/  ← 通用引擎，一行不改
  │
  └── adapter（你的领域知识薄层）← 你只需要写这个
```

**你只需要写一个 adapter，实现 5 个方法。** 引擎通过这 5 个方法理解你的领域。

### 5 个方法（以"代码生成"为例）

| 方法 | 作用 | 代码生成的例子 |
|---|---|---|
| `classify(candidate)` | 把候选分成"家族" | 按代码模式分：递归 / 迭代 / 函数式 / 面向对象 |
| `extract(candidate)` | 提取特征（用于去重+新颖性） | 提取：用了哪些 API、控制流结构、数据类型 |
| `motif_of(candidate)` | 预判失败模式（一类容易一起失败的候选） | "无 null 检查的可空类型访问" → motif: `null_unsafe` |
| `dead_end_reason(candidate, ctx)` | 判断是否已知死路 | 这个代码模式上次编译失败了 → 跳过 |
| `is_success(result)` / `is_weak(result)` | 评判结果 | 编译通过+测试全过 = success；编译过但测试部分失败 = weak |

### 接入代码骨架

```python
from core.engine import Engine
from core.memory import Memory

# 1. 写 adapter（你的领域知识，5 个方法）
class CodeGenAdapter:
    def classify(self, candidate):      # → "recursive" / "iterative" / ...
    def extract(self, candidate):       # → ["api:requests", "flow:for", ...]
    def motif_of(self, candidate):      # → "null_unsafe" 或 ""
    def dead_end_reason(self, candidate, ctx):  # → "already_failed" 或 ""
    def is_success(self, result):       # → tests_passed
    def is_weak(self, result):          # → compiled_but_warnings

# 2. 定义你的失败模式规则（哪些 motif 积累够多次就惩罚）
RULES = [
    RuleSpec("null_unsafe", "compile_error", 3, "penalize", -0.5,
             "无 null 检查的访问反复触发编译错误"),
]

# 3. 接入引擎（这一步和领域无关，照抄）
engine = Engine(
    adapter=CodeGenAdapter(),
    memory=Memory(),
    rule_specs=RULES,
)

# 4. 每轮搜索调用 engine.run()
result = engine.run(
    candidates=your_generated_code_variants,  # 你的 Agent 生成的候选
    universe="python_project_A",              # 环境标识
    execute=lambda code: run_and_test(code),  # 你的执行+评判函数
    max_execute=5,                            # 每轮最多试几个
)
# 引擎自动：屏蔽死路 → 排序 → 执行 → 记录 → 检查是否该换结构
```

### 接入工作量

| 领域 | adapter 行数 | 备注 |
|---|---|---|
| mock（玩具） | 54 行 | 最小参考实现 |
| game2048 | 482 行 | 含游戏环境 |
| symreg | 758 行 | 含 AST 求值器 |
| regex | 608 行 | 含 F1 评估 |

**纯 adapter 部分（不含环境/评估器）通常 50-150 行。** 环境和评估器是你项目本来就有的，痕进不要求你重写。

## 四、价值总结

| 维度 | 没有痕进 | 有痕进 |
|---|---|---|
| 第 1 次失败 | 记不住，下次还会犯 | 写进账本 |
| 第 5 次同类失败 | 还在犯 | 已被规则屏蔽，自动跳过 |
| 搜索方向 | 碰运气 | 往成功过的方向深化 |
| 可解释性 | "Agent 好像学到了" | 每条惩罚追得到具体哪几次失败教的 |
| 跨轮复利 | 每轮从零开始 | 记忆累积，越搜越准 |
