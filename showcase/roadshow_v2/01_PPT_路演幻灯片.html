THE PROMISE

失败留痕，
凭痕而进。

痕进不是让 Agent 背更多上下文，而是让它学会长记性：错在哪里、为什么错、下次该避开哪条路，都留下可追溯的痕迹。

THE PROBLEM

今天的 Agent，太容易反复犯同一个错。

客服、代码、搜索、研究都一样：一次失败修掉了，下一次换个壳又回来。问题不只是记忆不够，而是失败没有进入决策系统。

只记成功正确答案进了记忆库，错误路径却被当成垃圾清掉。
只调参数prompt 改来改去，但真正该换的是任务结构和搜索路径。
只追最优大家挤向看起来最好的方向，新路线还没被试就被放弃。
乔木计划 · failure becomes signal03 / 14
THE THESIS

不是修 prompt。
是让失败变成系统资产。

一次失败如果只停在对话窗口里，就是噪音；写进账本、归成模式、投影成规则、影响下一轮搜索，它才开始产生复利。

01

记住失败

失败持久化，别把配额烧在已知死路上。

02

评分搜索

评估“怎么找答案”，而不只评估最后答案。

03

学习模式

从“这次错了”升级到“这类情况都会错”。

04

换结构

方法反复失败时，换路径，不在旧路上微调。

05

避拥挤

奖励新颖探索，防止集体挤进局部最优。

OBSERVABLE EVOLUTION · PUBLIC DOMAIN

公开领域里，
进化轨迹可观察。

这页不证明“统计显著提升”，只回答一个更具体的问题：Agent 能不能从成功线索里造出新候选？在符号回归和正则合成里，每一轮结构变化都可观察、可追溯。

symreg

符号回归 · 发现隐藏公式

0.37(x*x)
→ 嫁接 ×2 →
0.9962*(x*x)

隐藏目标 y=2x²+1。R² 从 0.37 跃升到 0.996，公式从 x² 长出 2x²。

regex

正则合成 · 归纳分类规则

0.58.*
→ 收紧替换 →
0.74[0-9]+-[0-9]+

隐藏目标：识别 YYYY-MM-DD 日期。F1 从 0.58 升到 0.74，规则从过宽收紧到精确。

两种进化方向，同一个机制符号回归是加性生长（公式越长越接近），正则合成是收紧替换（规则越收越精确）。现场动画 → demo/08

DESIGN DECISION · NO LLM DEPENDENCY

引擎零 LLM 依赖。
纯本地、确定性、可复现。

引擎做的事是结构化的记录与检索——记住失败、屏蔽死路、给家族打分、投影规则。全是确定性的数据结构操作，不调任何大模型。

依赖 LLM 的"自进化"
每轮搜索成本要调 LLM，贵
可复现性LLM 输出不确定
可审计性"学到了"是黑箱
嵌入成本要带 LLM 才能跑
痕进（结构化进化）
每轮搜索成本零 API，免费
可复现性确定性，同输入同输出
可审计性每条规则有 evidence_ids
嵌入成本纯 stdlib，离线跑
LLM 负责造，引擎负责筛和记在你的项目里，LLM 可以作为候选生成器——但引擎本身的记忆、屏蔽、规则投影全部是确定性操作，不依赖任何大模型。

MORE ADAPTERS · SQL / CODE / RAG

更多场景能接入，
但别把跑通说成泛化证明。

SQL、代码生成、RAG 都跑出了可观察的修复链，说明 adapter 边界能迁移到不同任务。它们是接入证据，不是“普遍有效”的统计证明。

接入只写一个 adapter，
引擎一行不改。

OPEN CONTROL · AMNESIC vs FULL

预注册对照实验：
机制可观察，增益未达显著。

在公开的 2048 上做了预注册的失忆组 vs 完整组对照（M=30 配对）。唯一变量：Memory 是否跨 round 累积。结果 p=0.079——机制作为代码可观察地工作，但学习增益未达统计显著。符号回归和正则合成展示了可观察的进化轨迹，但同为种子化轨迹、非统计证明。

HONEST BOUNDARY

能做到什么，不能做到什么。

我们 claim

  • 5 个机制都已实现。
  • 观察到一次真实换结构事件。
  • 部分学习来自账本投影，不是手写规则。
  • 符号回归 + 正则合成：公开可复现的生成器进化轨迹。
  • SQL / 代码 / RAG：已接入并跑出可观察修复链。

我们不 claim

  • 没有普遍有效性的统计证明（2048 对照已跑，结果 null）。
  • 没有量化收益曲线。
  • 7 个领域 ≠ 泛化证明；接入跑通不等于性能显著提升。

区分「机制存在」与「机制有效」——前者有代码与对照证据，后者尚未被统计证明。这是项目的边界。
