# 路演文案 · 痕进（Scarforge）通用版

> 时长：5–7 分钟  
> 节奏：痛点 → 解法 → 产品 → 场景 → 壁垒

---

## 一、开场：一个反直觉的问题

各位好。

今天大家都在做大模型 Agent，但几乎所有 Agent 都面临同一个尴尬：**它会反复犯同一个错误。**

你写了一个客服 Agent，它在第 3 次对话里踩了坑；修复了。第 30 次，类似的场景，它又掉进去了。

你写了一个代码 Agent，同样的类型错误，它能犯十遍。

为什么？因为今天的 Agent 大多只有“记忆成功经验”的惯性，却没有一套**系统化的失败进化机制**。

我们做了一个不一样的尝试，项目代号 **痕进（Scarforge）**。

含义很简单：**失败留痕，凭痕而进。**

---

## 二、痛点：Agent 不会“长记性”

当前 Agent 的进化停留在三个很浅的层次：

1. **只记成功，不记失败**——向量库里塞满了正确答案，但错误答案被直接丢弃。
2. **只调参数，不换结构**——prompt 改来改去，但解决问题的方式本身就没对过。
3. **只追最优，不避拥挤**——大家都往同一个方向调，越容易收敛到越差的局部最优。

结果是：Agent 看起来很聪明，其实很脆弱。环境一变，立刻失效。

---

## 三、我们的解法：五个进化原则

痕进的核心不是某一个模型，也不是某一条 pipeline，而是**五个经过提炼的进化原则**：

**第一，记住你的失败。**

失败不是噪音，是信号。我们把每一次失败都结构化成可检索、可复用的记忆。

**第二，评分搜索，而不是候选人。**

不要只给最终答案打分。答案好不好，往往取决于前面的搜索路径。我们评估的是 Agent 怎么找到答案，而不是它最后给了什么。

**第三，学习失败模式，而不是单次失败。**

一次报错不重要，一类报错才重要。我们让 Agent 从“这次错了”升级到“这类情况都会错”。

**第四，换结构，不调参数。**

当一种方法反复失败，真正该改的是策略结构，而不是在旧结构上调 temperature、改几个词。

**第五，寻求新颖，避免拥挤。**

进化需要多样性。我们主动奖励那些探索新区域、避免扎堆的尝试，防止 Agent 集体陷入局部最优。

这五个原则，构成了痕进的底层方法论。

---

## 四、产品形态：一个轻量、通用、可嵌入的进化内核

痕进现在的实现非常轻量。

核心只有几个 Python 模块：引擎、进化器、记忆、元学习、新颖度计算、规则协议。

前端是四个纯静态 HTML 演示，不需要安装，不需要外部依赖，打开浏览器就能看到“失败如何被记录、结构如何被进化、决策如何被审计”。

但它不是玩具。它的设计目标是**成为任意 Agent 系统的进化层**。

你可以把它接在 LangChain 后面，接在 LlamaIndex 后面，接在你的代码 Agent、客服 Agent、研究 Agent 后面。它不关心你做什么任务，只关心一件事：**让你的 Agent 越失败，越聪明。**

---

## 五、应用场景：凡是会试错的 Agent，都能用

这里举几个最自然的场景：

- **代码生成 Agent**：编译错误、测试失败 → 自动归类失败模式 → 调整代码结构或提示策略。
- **自动化工具 Agent**：处理表格、调用 API 失败 → 记住“这类输入容易触发什么问题” → 下次换条路走。
- **搜索与信息检索 Agent**：搜索结果差 → 评估搜索路径 → 进化查询结构，而不是换几个关键词。
- **教育辅导 Agent**：学生反复出错 → 识别错误模式 → 调整讲解结构。
- **游戏与模拟 Agent**：在可重复环境里试错 → 积累经验 → 进化策略。

我们不绑定任何垂直行业。我们的定位是**通用进化基础设施**。

---

## 六、竞争优势：轻量、通用、方法论驱动

和市面上一些很重的垂直 Agent 平台相比，痕进的优势很明确：

- **更通用**：不是为某个行业定制的方法，而是一套跨领域的失败进化框架。
- **更轻量**：代码少、依赖少、可嵌入，不会给你的系统增加沉重负担。
- **更底层**：我们不替代你的 Agent，我们让你的 Agent 自己会进化。
- **可审计**：每一次失败、每一次结构变化、每一次进化决策，都留痕可查。

---

## 七、结尾：我们的愿景

大模型让 Agent 能说话、能调用工具、能写代码。

但真正的智能，不在于第一次就做对，而在于**第一次做错之后，第二次不再错**。

痕进要做的，就是让每一个 Agent 都能从自己的伤疤里长出新的能力。

**失败留痕，凭痕而进。**

这就是痕进。谢谢大家。
