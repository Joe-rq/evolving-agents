目标一：R² 跨轮轨迹 — 完整组 vs 基线组

隐藏目标 y = 2·x² + 1 （seed=3）。完整组生成器读 Memory、嫁接成功公式；基线组生成器与记忆无关。两组各自累积 Memory（屏蔽/规则都生效），唯一差别是生成器是否学习。

关键观察：完整组从 r3 起连续 5 轮 R²≥0.98（稳定守住）；基线组全程 ≤0.63，最后跌到 0.02（撞中又忘，反复震荡）。两者 r1 起点相同（0.37），分化完全来自生成器是否从成功里学习。

一条进化链：从 (x*x) 到 (2*(x*x))+3

r1 随机池出现 (x*x)（R²=0.37，正向线索）。生成器随后围绕它嫁接：先试系数，再试常数项。每一步都是结构可见的新公式。
+0.370 (x*x) ← r1 随机池，poly 族正向线索
↓ r3：生成器读 Memory，发现 poly 成功 → 嫁接 ×2（探索系数）
+0.996 2*(x*x) graft: ×2 → R² 从 0.37 跃升到 0.996
↓ r4：在成功根上嫁接 +3（探索常数 offset）
+0.983 (2*(x*x))+3 graft: +3（offset 探索）
↓ r6：尝试给成功公式嫁接 sin 项（探索混合族）
+0.979 ((2*(x*x))+3)+sin(x) graft: +sin(x)（跨族探索，R² 略降但仍高位）
这就是 2048 给不了的进化：每一步是结构可见的新公式（不是权重 ±0.05），R² 从 0.37 跃升到 0.996，血统完整可追溯。最终产物 2*(x*x) 人类可读——它不是黑箱"学到了"，是写出来的公式。

目标二：更难的混合目标 — 证非 cherry-pick

隐藏目标 y = x² + sin(x) （seed=7）。混合 poly + trig 两族——更难，因为 agent 要同时发现两个方向并组合。换目标 + 换 seed，排除"只挑了好跑的"。

r1-3((2*x)+(-1)²)…+0.51→0.59-0.42→0.99围绕 linear 线索嫁接，R² 卡在 0.6（错误方向）
r4(x*x)+0.99+0.89poly 族突破，R² 跃升

r7((x*x)+-2)+-2+0.64-0.00完整组仍在 0.64；基线组归零
两个目标，同一种模式：完整组能放弃错误线索（r1-3 围绕 linear 嫁接，发现卡在 0.6，r4 转向 poly），然后围绕新成功根继续爬升。基线组则在两个目标上都表现"撞中又忘、最终崩盘"。这排除了 cherry-picking——换目标、换 seed，模式一致。

诚实边界：这证明了什么、没证明什么

✅ 证明了⚠️ 没证明
生成器读 Memory 后，能从成功线索嫁接出新公式；R² 跨轮稳定在高位。两个目标（seed 各异）模式一致 → 非单例。 这不是 M-复制对照实验——是 2 条种子化轨迹。统计显著的"进化增益"未被证明。
进化是结构可见的：每步是新 AST 公式，血统可追溯到父代 + 嫁接操作。最终产物人类可读。 不能 claim "进化提升 X%"——轨迹级证据，且隐藏目标固定（虽已换 2 个目标证非 cherry-pick）。
引擎一行未改接入第三个领域（2048 → 迷宫 → 符号回归）。 跨领域泛化仍是"3 个领域 ≠ 普适证明"——和诚实边界一贯立场一致。
为什么 nonetheless 值得讲：大多数"自进化 agent"demo 给你一条学习曲线，却回答不了"这是 agent 学的还是 in-context learning？是结构变化还是参数抖动？"。这个案例至少让前者可审计——你能看到每一次嫁接、能追溯每个公式的父代。这是黑箱曲线系统没有的。
