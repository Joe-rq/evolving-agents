evolving-agents

痕进

失败留痕，凭痕而进。
造能从自己的失败里学习的 agent。

大多数搜索循环不会学习。 要么每次从零开始，要么把历史塞进 prompt 碰运气。真正会进化的 agent 做的事不一样：给搜索策略打分，学习失败模式而非单个失败，卡住时换结构而非调参数。

五个 Factor
Factor 1

记住你的失败

每次搜索留下一条疤痕记录：失败表达式、失败原因、所在池塘。下次不重复烧配额。

memory · rules
Factor 2

给搜索打分，不是给候选打分

agent 评价的不是"这个候选好不好"，而是"这次搜索过程值不值得复用"。搜索策略本身才是学习对象。

meta_learner
Factor 3

学失败模式，不是单个失败

一次 self-corr 失败后，整个 close-yield motif 类被降权——不只屏蔽那一条。从账本投影出规则，不是人写的规则。

rules · evidence_ids
Factor 4

换结构，不是调参数

在拥挤池塘里调 decay/窗口永远修不好问题。agent 提议换数据源、换搜索空间——人批准。

evolver · pivot
Factor 5

追新颖，避拥挤

novelty 模块追踪搜索过的空间，让 agent 在已有疤痕附近主动远离，探索新区域。

novelty · self_corr

诚实声明
⚠ 我们报告 null 结果

NULL 在 2048 领域的失忆组对照（M=30）中，Memory 跨 round 累积的增益 未被统计证明（p = 0.079）。

但我们同时报告：机制作为代码 全部可观察地工作——block 452 次、 pivot 131 次、规则投影 1.7 条，失忆组全部为零。我们区分「机制存在」与「增益被统计检出」，后者未成立即如实写入。这是这个项目的诚实边界。
